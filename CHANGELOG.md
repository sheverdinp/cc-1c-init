# CHANGELOG

Все значимые изменения в этом репозитории документируются в этом файле.

## 2026-04-12 — release_002: границы ФС/БД, параметры подключения, git-чекпоинты, двухшаговый workflow Claude→Codex

Добавлены жёсткие границы работы AI-агента (файловая система и одна база 1С из `.dev.env`), параметры подключения к базе, git-чекпоинты перед этапами задачи, второй промт `patch_codex.md` для портирования конфигурации в Codex и идемпотентный патч для обновления существующих проектов.

### Подробно

**CLAUDE.md (шаблон в `promt_new_project.md`)**

- Новая секция **«Границы файловой системы и БД»**: чтение/запись только в папке проекта; подключение только к единственной базе из `.dev.env` (`DB_SRVR`, `DB_REF`, `DB_USER`, `DB_PWD`). Разрешены операции через навыки `.claude/skills/` (cc-1c-skills) и штатный цикл разработки расширений: `DumpConfigToFiles`, `LoadConfigFromFiles`, `UpdateDBCfg`, загрузка/выгрузка расширений, `LoadExternalDataProcessorOrReportFromFiles`, `CheckModules`. Подключение к другим базам, произвольный код, массовые изменения данных, административные операции — только с явного подтверждения в сессии. Работа в продуктивной базе запрещена. `DB_PWD` запрещено логировать, выводить в отчётах и коммитить.
- Новая секция **«Git-чекпоинты»**: коммит-чекпоинты перед планированием / разработкой / тестированием в формате `T-XXXX чекпоинт перед <этап>`.

**`.dev.env.example` (шаблон)**

- Добавлен блок параметров подключения к базе 1С с примерами (`DB_SRVR`, `DB_REF`, `DB_USER`, `DB_PWD`). Напоминание о секретности `DB_PWD` и о том, что `.dev.env` не коммитится.

**`kb/1c-rules/coding-standards.md` (инструкция конвертации)**

- В секции «Extended Development Standards» ссылки `@rules/*.mdc` заменяются на указатели внутренних разделов файла (стандарты уже инлайнены при конвертации).
- Разрешён конфликт венгерской нотации: префиксы `м_`, `стр_`, `сп_` запрещены, но семантическое отражение типа в имени переменной (`МассивСчетов`, `СтруктураПараметров`) — обязательно.

**`check_up.md` (промт диагностики)**

- П.4 расширен: дополнительно проверяется `kb/1c-rules/mcp-tools.md` (существует и согласован с `CLAUDE.md`/`coding-standards.md`) и `kb/1c-rules/agents/*.md` (все файлы из таблицы ролей существуют, нет неучтённых).
- Новый п.7: проверка доступности MCP-серверов — вызов `stats` для каждого подключённого MCP-сервера со статусом «отвечает / не отвечает / ошибка».
- Новый п.8: план действий — для каждой найденной проблемы предлагается конкретное решение, которое можно выполнить прямо сейчас с помощью AI.
- Формат вывода: «список проблем и план действий» вместо «списка проблем и рекомендаций».

**Новый промт `patch_codex.md`**

- Второй шаг workflow после `promt_new_project.md`. Портирует Claude-конфигурацию проекта в project-local Codex-конфигурацию: `.codex/config.toml` (с перенесёнными MCP и `project_doc_fallback_filenames = ["CLAUDE.md"]`), `.agents/skills/` (порт navibков из `.claude/skills/`), `.codex/rules/project.rules` (безопасный перевод `permissions.allow` из `.claude/settings.local.json`), sync-скрипты (`scripts/sync-codex-mcp.ps1`, `sync-codex-skills.ps1`, `sync-codex-rules.ps1`), `check_up_codex.md`, отчёт `meta/codex-port-report.md`.
- Codex-rules не генерируются шире, чем разрешают границы в `CLAUDE.md` (нет широких `allow` для bash/python/powershell, нет обходов FS/DB boundaries).
- Не трогает `.dev.env` и не выводит `DB_PWD`. Не изменяет глобальные настройки пользователя.
- После применения самоудаляется из корня проекта.

**Новый патч `promt_patch_project_to_r002.md`**

- Идемпотентный патч для обновления проектов, инициализированных предыдущей версией промта. Обновляет `CLAUDE.md` (две новые секции), `.dev.env.example` и `.dev.env` (блок `DB_*`, существующие значения не перезаписывает), `kb/1c-rules/coding-standards.md` (правки Extended Development Standards и венгерской нотации), `check_up.md` (новый список проверок). Выдаёт отчёт по каждому шагу и самоудаляется.

**`README.md`**

- Описан двухшаговый процесс запуска: сначала `promt_new_project.md` в Claude Code, затем `patch_codex.md` в Codex.
- Новая секция «Границы работы AI-агента» с кратким описанием FS/DB boundaries.
- Упомянут патч `promt_patch_project_to_r002.md` для существующих проектов.
- В описании `.dev.env.example` добавлены параметры подключения к БД.
